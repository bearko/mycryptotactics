## 1. 概要

**分岐マップ → ターン制カード戦闘 → 報酬** の体験要件を定義する。実装の正は [SPEC-002](./SPEC-002-prototype.md) の `prototype/`。`mock/index.html` はプロトタイプへリダイレクトするに留める。アセットは [bearko/mycryptoheroes](https://github.com/bearko/mycryptoheroes) の raw URL を参照する。

## 2. 背景と動機

企画の手触り検証と Vercel 上での共有のため。詳細な世界観は [CHRONO_TACTICS_DESIGN.md](../CHRONO_TACTICS_DESIGN.md) を参照。

## 3. 目標と非目標

### 目標（Goals）

- デプロイ URL からプレイ用ページを閲覧できる。
- 対峙型の戦闘 UI と Backgrounds / Extensions 由来のカード表現が分かる。

### 非目標（Non-goals）

- セーブデータの永続化、バランスの確定、本番用アセット配布。

## 4. スコープ

### 含む

- マップ層のノード選択、戦闘、店、休憩、ゲームオーバー、ランリセット。
- 戦闘アリーナの背景画像（`Image/Backgrounds/`）。
- カードのエクステ画像・名称・スキル名・アイコン（データに基づく簡略実装）。

### 含まない

- サーバー API、認証、課金。

## 5. 契約と挙動

### 入力

- ユーザーのクリック（マップノード、カード、ターン終了、報酬、店、リスタート）。

### 出力・副作用

- DOM の更新、外部画像 URL への GET（ブラウザ）。

### エラー条件

- 外部画像が取得できない場合はブラウザ標準の壊れアイコンとなる（モックではフォールバック画像は必須としない）。

### 互換性

- ルート `index.html` は **プレイ用エントリ**（現行: `/prototype/index.html`）へ誘導し、Vercel で `/` が開けること。

## 6. 非機能要件

- **パフォーマンス**: 初回は画像サイズに依存（許容）。
- **ライセンス**: mycryptoheroes の利用条件を憲章・Human-in-the-loop に従い確認する。

## 7. Human-in-the-loop

- 外部アセットのライセンス変更・商用利用の可否判断。

## 8. 受け入れ基準

- [ ] プレイ用 URL（`prototype/index.html`）でマップから戦闘に入れる。
- [ ] 戦闘画面でリーダーと敵が左右対峙し、背景が表示される。
- [ ] 手札カードに Extension 画像とスキル名・効果説明が表示され、プレイで敵 HP が変化する。
- [ ] 戦闘勝利後に報酬選択またはスキップでマップに戻れる。
- [ ] ルート URL からプレイ用に到達できる（`index.html` のリダイレクト）。
