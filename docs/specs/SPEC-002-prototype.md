## 1. 概要

[SPEC-001](./SPEC-001-roguelike-deck-mock.md) の単一 HTML モックを **プロトタイプ** に移行する。`prototype/` に ES モジュールでロジックを分離し、企画書の「データ駆動設計」方針に沿ってカード・マップ定義をファイル分割する。戦闘画面の手札は **常に3列** とし、狭いビューポートでも5枚が概ね縦スクロールなしで確認できること。

## 2. 背景と動機

モック段階で体験が固まったため、マイルストーン「戦闘シミュレータ／マップの縦切り」に向けコード構造を整える。参照: [CHRONO_TACTICS_DESIGN.md](../CHRONO_TACTICS_DESIGN.md) §6・§8。

## 3. 目標と非目標

### 目標

- デプロイ先のルートからプロトタイプに到達できる。
- カード定義が `prototype/js/cards.js` に集約され、`main.js` が薄くなる。
- 手札 UI が **CSS Grid で3列** 固定。

### 非目標

- ビルドツールの導入、JSON フェッチによるランタイム読込（次 SPEC で検討可）。

## 4. スコープ

### 含む

- `prototype/index.html` + `prototype/js/*.js`
- `mock/index.html` をプロトタイプへの誘導に変更
- `index.html` のリダイレクト先を `/prototype/index.html` に変更

### 含まない

- 新カード種・新ノード種のゲームデザイン拡張（別 SPEC）。

## 5. 契約と挙動

- ゲーム挙動は SPEC-001 と同等（回帰として手動確認）。
- 手札コンテナ `.hand` は `grid-template-columns: repeat(3, 1fr)` とする。

## 6. Human-in-the-loop

- 外部アセット利用方針の変更時は [HUMAN_IN_THE_LOOP.md](../process/HUMAN_IN_THE_LOOP.md)。

## 7. 受け入れ基準

- [ ] `/prototype/index.html` でマップ〜戦闘〜報酬が動作する。
- [ ] ルート `/` および `/mock/index.html` からプロトタイプに誘導される。
- [ ] 戦闘中、手札5枚が **3列×2行**（上段3・下段2）で表示され、主要端末幅で縦スクロール不要に近いこと。
- [ ] `prototype/js/cards.js` にカードライブラリが存在し、`map.js` にマップ層がある。

## 8. テストと証跡

手動: ブラウザで上記 URL を開き、戦闘に入って手札レイアウトを確認。
