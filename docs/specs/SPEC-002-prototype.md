## 1. 概要

[SPEC-001](./SPEC-001-roguelike-deck-mock.md) の単一 HTML モックを **プロトタイプ** に移行する。`prototype/` に ES モジュールでロジックを分離し、企画書の「データ駆動設計」方針に沿ってカード・マップ定義をファイル分割する。戦闘画面の手札は **常に3列** とし、狭いビューポートでも5枚が概ね縦スクロールなしで確認できること。加えて **戦闘の視認性・フィードバック・情報の近接配置** を SPEC で固定し、以降の機能追加時の参照とする（§10）。

## 2. 背景と動機

モック段階で体験が固まったため、マイルストーン「戦闘シミュレータ／マップの縦切り」に向けコード構造を整える。スマホでの可読性・テンポ感を損なわない範囲で、数値変化と敵意図が一目で伝わる UI を求める。参照: [CHRONO_TACTICS_DESIGN.md](../CHRONO_TACTICS_DESIGN.md) §6・§8・§9（体験設計の柱）。

## 3. 目標と非目標

### 目標

- デプロイ先のルートからプロトタイプに到達できる。
- カード定義が `prototype/js/cards.js` に集約され、`main.js` が薄くなる。
- 手札 UI が **CSS Grid で3列** 固定。
- 戦闘中の **HP / PHY / INT / AGI / ガード / シールド** がリーダー・敵の双方で表示され、バフ系カードで変化が追える。
- **情報の近接配置**（エナジーは手札直上、敵の次行動は敵近傍、ログはアリーナと手札の間）。

### 非目標

- ビルドツールの導入、JSON フェッチによるランタイム読込（次 SPEC で検討可）。
- WebAudio での複雑なミキシング（現状は `<audio>` / `new Audio()` で十分）。

## 4. スコープ

### 含む

- `prototype/index.html` + `prototype/js/*.js`
- `mock/index.html` をプロトタイプへの誘導に変更
- `index.html` のリダイレクト先を `/prototype/index.html` に変更
- 戦闘 BGM（`Audio/BGM/pvp.mp3`）、ランクリア SE（`Audio/SE/clear.wav`）の再生
- 被ダメージ時のポートレートフラッシュ、数値変化のフロート（赤減／緑増）
- ドット絵の **ニアレストネイバー系** 拡大（`image-rendering: pixelated` / `crisp-edges`）

### 含まない

- 新カード種・新ノード種のゲームデザイン拡張（別 SPEC）。

## 5. 契約と挙動

- ゲーム挙動のコアは SPEC-001 と概ね同等（マップ〜報酬）。**戦闘ダメージ計算**は §11 の MCH ヘルプ準拠式に置き換えた（プロトの数値は調整可）。
- 手札コンテナ `.hand` は `grid-template-columns: repeat(3, 1fr)` とする。
- **ヘルプ**: 画面上部左の「?」でオーバーレイを開き、目的・操作・パラメーター・バトルをタブ状に切り替えて表示。`Esc` または背景クリックで閉じる。
- **フッター**: 技術スタックの短い注記（仕様駆動・データ分離・手札列数・ピクセル補間方針）を表示してよい。
- **音声**: 戦闘 `showView('combat')` で BGM ループ開始、戦闘終了で停止。ラン最終層クリア（マップ上でクリア表示）時に SE を 1 回再生。
- **戦闘 SE**（[Audio/SE/Battle](https://github.com/bearko/mycryptoheroes/tree/main/Audio/SE/Battle)）: 単体ダメージ `1_single_damage`、範囲/特殊系 `2_area_damage`、回復 `3_heal_resurrection`、バフ `4_buff`、デバフ `5_debuff_status_effect`。攻撃ヒット・被ダメ時は単体 SE、敵の特殊意図で範囲 SE、回復・バフ・デバフは各 SE。
- **NEXT ACTION**: ラベルは `NEXT ACTION`。本文は `ATK PHY n%` / `GUARD +n` / `SPECIAL maxHP n%`（敵意図）。表示位置は **敵ポートレート直下**。
- **エナジー**: 手札グリッド直上の `.hand-energy-row` に表示。
- **ログ**: `#clog` は **アリーナと手札エリアの間**（`.combat-log-mid`）に配置。スマホでもスクロールなしで直近ログが見えることを優先。
- **数値ポップ**: HP・ガード・シールド・PHY/INT/AGI の増減を `.stat-anchor` 付近に一時表示（減は赤系、増は緑系）。

- [ ] ログが **アリーナと手札の間** にあり、スマホで主要操作ゾーン近くに見えること。

- [ ] HP/ガード/シールド/ステの変化で **±数値フロート**（減赤・増緑）が出る。

## 10. 体験・UI 設計方針（機能追加時の参照）

以下は **意図の固定**であり、変更する場合は本 SPEC の改訂と PR 説明に理由を書くこと。

1. **情報密度と優先度**: 戦闘中は「盤面（対峙）→ ログ → 手札」の縦の流れで視線が止まるようにする。装飾用の大きなヘッダーは避け、同一画面内に戦闘に必要な情報を収める。
2. **近接配置**: プレイヤーが結びつけたい情報同士を近づける（例: エナジーと手札、敵の意図と敵キャラ、ログと直前の行動結果）。
3. **モバイルファースト**: 縦スクロールに依存して初めて状況がわかる配置にしない。ログは親指エリア付近に置く。
4. **フィードバックの階層**: （a）即時アニメ・色・**SE**（ヒットフラッシュ、数値ポップ）→（b）ログのテキスト。テンポを落とさずに「何が起きたか」を補強する。
5. **用語の明瞭化**: 敵専用の UI には英字ラベル `NEXT ACTION` 等で役割を明示し、初見でも対象が敵であると分かるようにする。
6. **数値の検証可能性**: バフ・デバフで変わる主パラメータ（HP / PHY / INT / AGI / ガード / シールド）は戦闘中に常時可視とし、カード選択の結果を検証しやすくする。
7. **音声**: BGM はループ、SE はワショット。ファイルはリポジトリ同梱の `Audio/` を参照し、音量は控えめから調整可能にする。
8. **ピクセルアート**: ソース解像度が低いスプライトは **バイキュービック等で拡大しない**（ぼやける）。ブラウザでは `image-rendering: pixelated` / `crisp-edges` により **ニアレストネイバー系** のみを用いる。別アルゴリズムが必要な場合はプリレンダまたはシェーダを別 SPEC で検討。
9. **SPEC ファースト**: 上記の変更は「体験要件」として SPEC-002 に残し、実装はこれに追従する。逸脱は SPEC 更新か PR で明示する。

## 11. MCH 準拠の戦闘数式（プロトタイプ）

原作 [My Crypto Heroes](https://www.mycryptoheroes.net/) のヘルプに記載の式をプロトで再現する。一次ソースは以下（日本語ページ。英語版と同一内容）。

- [パラメーターの役割](https://www.mycryptoheroes.net/ja/help-posts/7iMLBLL1sMJ9Rlk2Fv2Zmj) — 回復係数 `(術者INT+対象PHY)/2`（INT 回復）、`(術者PHY+対象INT)/2`（PHY 回復）
- [ダメージとカット率](https://www.mycryptoheroes.net/ja/help-posts/11ek6KSt0fq0x5gdUeKlie) — PHY 攻撃のカット率 `min(40, floor(対象PHY/2))`%、INT 攻撃は `min(40, floor(対象INT/2))`%。PHY/INT 依存ダメージ `floor(攻撃側ステ×スキル%/100 × (100-カット率)/100)`
- [クリティカル](https://www.mycryptoheroes.net/ja/help-posts/3HfmOjuWDSyw1isnhnLSfv) — 発生時、上記と同様の計算で **バフデバフ無視の戦闘開始時基礎値** で追加分を算出し合算
- [シールド値](https://www.mycryptoheroes.net/ja/help-posts/2mNToh5xAynqPi6V23S9xc) — PHY/INT 依存ダメージには無効。**特殊ダメージ**（最大 HP 割合など）のみシールドが先に吸収。超過分は HP へ
- [バフとデバフ](https://www.mycryptoheroes.net/ja/help-posts/6D3DPoIZhiJqPy4eYDF62O) — PHY/INT/AGI は 1〜999、HP は最大 1000（プロトは小規模のため上限未到達を想定）

### 11.1 本作プロトでの差分・拡張

- **チャージ廃止**: 公式の AGI とチャージの関係は本プロトでは採用しない。
- **AGI → クリティカル率**: `critRate = min(45, floor(AGI/2))`（%）。PHY 依存攻撃は PHY クリ率、INT 依存攻撃は INT クリ率として **同一式で AGI を流用**（ターン制への合意。原作の PHY/INT 別クリ率へ分離する場合は別 SPEC）。
- **ガード（Guard）**: ヘルプの「PHY/INT 依存ダメージに対する軽減」に相当する **固定値バリア**をプロトで実装（ターン開始時に味方ガードは 0 にリセット。敵ガードは意図で付与）。
- **シールド（Shield）**: 上記ヘルプ通り **特殊ダメージ**にのみ適用（敵意図 `SPECIAL maxHP n%` 等）。

実装ファイル: `prototype/js/battle-mch.js`（純関数）、`prototype/js/main.js`（適用とログ）。
