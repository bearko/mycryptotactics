## 1. ゴールの再定義

要求は「Vercel で稼働させて全ステージ遊べる状態で一般解放」。これを噛み砕くと、**最低限こうなっていれば公開できる** という線を引く必要があります。

### 1.1 最小成功条件（MVP for Launch）

- 章 1 → 章 2 → 章 3（ボス）まで通しでクリアできる。途中で進めなくなる致命バグがない。
- スマホ（iPhone Safari / Android Chrome）と PC（Chrome / Firefox / Safari）で破綻なく描画される。
- ヘルプ（? ボタン）からゲームの目的・操作・パラメータ・バトル数式を読める。
- アセット読込失敗時に「真っ白で固まる」が起きない（壊れアイコンで継続）。
- README にプレイ用 URL・操作・既知の制約が書かれている。

### 1.2 やらないことの明示

リリース時点では**しない**ことを書いておきます。後で「あれは？」とならないように。

- セーブ・進捗保存（localStorage すら使わない）
- アカウント／ログイン／ランキング
- BGM のミキシング・音量設定 UI（音量はブラウザに委ねる）
- 多言語対応（日本語のみ）
- 新しいカード・敵・ボスのスプライト追加（既存素材から流用）
- バランスの「正解」を出す（係数調整のみ）
- E2E 自動テスト（手動 QA で代替）

これらは SPEC-005 以降に倒す。

## 2. クリティカルパス

ステージ設計が「これから」というのが最大のリスクなので、**章コンテンツが最優先**。それ以外は順位付けして「これより下は落ちる」と先に決めておきます。

### 優先度（高 → 低）

1. **SPEC-004（章コンテンツ）の確定とデータ投入**：これがないと「全ステージ遊べる」を満たせない
2. **章 3 ボス戦の動作確認**：1 戦闘でも詰まる箇所があるとリリース不可
3. **モバイル実機チェック**：iOS Safari の音声自動再生ブロックや視認性の最終確認
4. **致命バグ修正**：HP 0 でゲームオーバーが発火しない、無限ループ、画像 URL の 404 等
5. **README 整備とプレイ URL の表記統一**
6. **SE/BGM の最終調整**（音量・ループ範囲）
7. **デザイン微調整**（フォント、余白、色）
8. **演出強化**（カットイン、エフェクト追加）

優先度 6 以下は **D+3 朝の時点で時間が余っていたら触る** 扱い。

## 4. ステージ／コンテンツの設計方針

ステージ＝章という単位で 3 つに分け、章ごとに「ノードグラフ」「敵プール」「カードプール」「ボス」を持たせます。詳細は [SPEC-004](../specs/SPEC-004-stage-content.md) に書いてありますが、骨子だけここに：

- 章 1：**戦国回廊**（4 層 × 3〜4 ノード）— 操作と戦闘数式を体に入れさせる。敵の意図はシンプルな ATK/GUARD のみ
- 章 2：**大航海の港**（5 層 × 3〜5 ノード）— 状態異常（毒・出血）の導入。ショップ・休憩イベントを増やす
- 章 3：**決定の街**（5 層 × 3〜5 ノード）— ラスボス。SPECIAL（最大 HP %）を使ってシールドの存在意義を出す

3 章 × 平均 4 層 × 平均 3 ノード ≒ **36 ノード**。1 ラン 30〜45 分を想定。
