# クロノ・タクティクス（仮）— 歴史上の偉人が戦う戦略ゲーム 企画書

## 1. コンセプト

**一文**: 古今東西の歴史人物を「将」として編成し、地形と相性を読みながら盤上で勝つ、ターン制の戦略シミュレーション。

**体験の核**

- プレイヤーは「時代を超えた軍議室」の司令官として、偉人カードを部隊に配し、マップを支配する。
- 各偉人は実在の業績・性格・時代背景をモチーフにしたスキルとステータスを持ち、単なる数値ではなく「この人ならこう動く」という納得感を目指す。
- 勝利条件はマップごとに変化（占領、護衛、撃破、資源争奪など）し、リプレイ性を担保する。

**ビジュアル・アセット方針**

- キャラクター立ち絵・一部 UI アイコンは [bearko/mycryptoheroes](https://github.com/bearko/mycryptoheroes) リポジトリの `Image/` および `Data/` を参照して利用する（ライセンス・利用条件はリポジトリの記載に従う）。
- 本モックは同リポジトリの `raw.githubusercontent.com` 経由で画像を読み込み、レイアウトと情報設計の検証用とする。

## 2. ターゲットとプラットフォーム

| 項目 | 想定 |
|------|------|
| 主ターゲット | 歴史・偉人に興味がある 20〜40 代、SRPG・デッキ構築に慣れた層 |
| 副ターゲット | ピクセルアート・レトロ調 UI が好きな層 |
| プラットフォーム（初期） | Web（PWA 想定）または PC 向け小売り風ダウンロード |
| セッション長 | 1 マップ 15〜25 分を目安（途中セーブ必須） |

---

## 3. コアゲームループ

1. **編成**: 所持している偉人から最大 N 体を選び、装備・「思想」（後述）を割り当てる。
2. **出撃**: マップを選び、勝利条件を確認する。
3. **ターン戦**: 味方ターンで移動・攻撃・スキル・待機を選択。敵 AI が応戦。
4. **裁定**: ターン終了時に地形効果・ドットダメージ・勝敗条件をチェック。
5. **報酬**: 通貨・経験・新規偉人の断片などを獲得し、編成に還流。

---

## 4. 主要システム

### 4.1 盤面・移動

- **タイル**: 平地・森（回避補正）・高地（射程・視界）・水域（特定ユニットのみ）など。
- **移動コスト**: ユニットの「機動力（AGI 由来）」でタイルごとに消費。ゾーンオブコントロール（敵隣接マス通過ペナルティ）をオプションで検討。

### 4.2 戦闘解決

- **近接 / 遠隔 / 計略** の 3 系統。命中率は距離・地形・デバフで変動。
- **パッシブ**: HP 閾値・ターン開始・撃破時など条件付き発動（参照データのヒーロー定義のイメージに近い）。
- **アクティブスキル**: クールダウンまたは「士気」消費で使用。

### 4.3 偉人の属性（例）

参照データに倣い、**HP / PHY（武力）/ INT（知略）/ AGI（機動）** を基本パラメータとし、UI では [BattleIcons のパラメータアイコン](https://github.com/bearko/mycryptoheroes/tree/main/Image/BattleIcons/Parameters) で表示する。

- **時代タグ**: 古代・中世・近世・近代 など（シナジー・ストーリー分岐）。
- **思想タグ**: 保守・革新・中庸（PvE ストーリーでの選択肢と連動可能）。

### 4.4 メタ進行

- 偉人のレベル上限解放、新スキル開放、図鑑（史実エピソードの短文）。
- 難易度別マップと週次チャレンジでエンドコンテンツを用意。

| 柱 | 内容 |

| スカーミッシュ | ランダムマップまたは固定チャレンジ。 |

## 6. 技術・データ（モックとの対応）

- **静的モック**: `mock/index.html` — 盤面選択・移動・簡易攻撃の UI フローと、アセット URL の検証。
- **本実装時**: 偉人マスタは JSON / DB で管理し、`heroes.json` と同様の構造（id, 多言語名, image_file_path, stats）を踏襲すると移植が容易。

---

## 7. リスクと未決事項

- **ライセンス**: mycryptoheroes データの二次利用可否はリポジトリ README・ライセンスファイルで必ず確認すること。
- **表現**: 実在人物の描写は各国のガイドライン・配信プラットフォーム規約に合わせてトーンを調整。
- **未決**: マップ形状（六角 / 四角）、同時攻撃の有無、ガチャの有無はプロトタイプの手触りで決定推奨。
