# MyCryptoHeroes（元タイトル）ロアリファレンス

> **このドキュメントの位置づけ**
> 派生作 *MyCryptoTactics* に「元タイトルプレイヤーがニヤッとする小ネタ」を企画／実装する際の知識ベース。
> 新しい小ネタを足す前に **§9 Implementation Map**（実装済リスト）と **§10 未消費の有望ネタ** を必ず確認すること。
> 命名・テキストフレーバー単独より、**プレイヤーの選択や戦闘ループに絡む仕掛け** に翻訳した方が体験として強い（実証済）。

---

## 1. 元タイトル概要

- **名称**: My Crypto Heroes（マイクリプトヒーローズ／略 *マイクリ*／*MCH*）
- **運営**: 当初 double jump.tokyo → 2021/9 に **MCH 株式会社** 設立で独立
- **リリース**: 2018/9 バトルβ → 2018/10/18 正式リリース
- **キャッチコピー**: 「時間も お金も 情熱も、あなたの資産となる世界」
- **特徴**: ブロックチェーンゲーム（BCG）／オンチェーン資産＋オフチェーン戦闘記録／2018/10 OpenSea 取引量で世界 1 位
- **エコシステム世代**:
  - **Eco 1.0** Lab／GUM（〜2020）
  - **Eco 2.0** ランド／TOKU（2019/3〜2021）
  - **Eco 3.0** MCHC（2021/9〜）／MCH Verse（2022/12〜）

## 8. 採用ガイドライン（重要）

派生作 *MyCryptoTactics* に小ネタを足すときの判断基準：

1. **プレイの意味が変わるか**を優先
   - ✅ 選択肢／戦闘ループ／デッキ構築に絡む
   - △ 命名・テキストフレーバー単独
   - 命名だけでも入れて良いが、それ単体だけで PR を組むのは弱い
2. **既存スロットへの差し替え** で実装できるか確認
   - 章ノードの type／ボス／カード／ext は既存システムを再利用
   - 新ビューが必要なものは index.html の panel を 1 個追加（参考: `eventView` `scoutView`）
3. **アセット**は `bearko/mycryptoheroes/Data/Extensions` のものを **流用**（imgId / extId）
   - 新規アート作成は避ける／アセットは MCH 社所属
4. **Phase 1 で消費したネタを重複しない**（§9 を参照）
5. **コアファンが知らないと意味不明なネタは避ける**か、フレーバーテキストに最小限の説明を添える
